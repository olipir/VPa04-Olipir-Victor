## День 1. Что такое SQL и реляционные базы данных

**Что изучим:**
- Понятие базы данных (БД) и системы управления базами данных (СУБД).
- Реляционная модель: таблицы, строки (записи), столбцы (поля).
- Что такое SQL и зачем он нужен.
- Основные диалекты (PostgreSQL, MySQL, SQLite) и наш учебный стандарт.

**Ключевые мысли:**
База данных — это организованное хранилище информации. Реляционная БД представляет данные в виде связанных таблиц (как листы Excel). SQL (Structured Query Language) — язык, с помощью которого мы общаемся с базой: извлекаем данные, добавляем, изменяем, удаляем и управляем структурой.

*Пример таблицы `users`:*

| id | name   | email              |
|----|--------|--------------------|
| 1  | Иван   | ivan@example.com   |
| 2  | Мария  | maria@example.com  |

**Контрольные вопросы:**
1. Что такое реляционная база данных?
2. Из каких элементов состоит таблица?
3. Как расшифровывается SQL и для чего он нужен?
4. Назови хотя бы две популярные реляционные СУБД.

## День 2. Основная команда: `SELECT ... FROM`

**Что изучим:**
- Как получить данные из таблицы: оператор `SELECT`.
- Выбор конкретных столбцов и всех столбцов (`*`).
- Ключевое слово `FROM` для указания таблицы.
- Базовый синтаксис и точка с запятой в конце.

**Ключевые мысли:**
`SELECT` — самая важная команда. Она не изменяет данные, только читает. Ты говоришь базе: «Выбери мне эти столбцы из вот этой таблицы».

```sql
-- Выбрать все столбцы из таблицы users
SELECT * FROM users;

-- Выбрать только имена и почту
SELECT name, email FROM users;
