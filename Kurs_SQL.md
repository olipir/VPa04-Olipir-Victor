## День 1. Что такое SQL и реляционные базы данных

**Что изучим:**
- Понятие базы данных (БД) и системы управления базами данных (СУБД).
- Зачем нужны базы данных и чем они лучше Excel.
- Реляционная модель данных: таблицы, строки, столбцы.
- Типы связей между таблицами (один-к-одному, один-ко-многим, многие-ко-многим) — на уровне концепции.
- Что такое SQL, его назначение и стандарты.
- Краткий обзор популярных СУБД (PostgreSQL, MySQL, SQLite, SQL Server).

**Ключевые мысли:**
Реляционная БД — это не просто набор таблиц, а система со строгими правилами. Каждая таблица представляет сущность (например, «Пользователи», «Заказы»). Строка — это экземпляр сущности, столбец — свойство. Связи между таблицами реализуются через ключи. SQL — декларативный язык: ты описываешь *что* хочешь получить, а не *как* это сделать.

*Пример: таблицы `users` и `orders` связаны как «один-ко-многим» (один пользователь может иметь много заказов).*

**Контрольные вопросы:**

2. Из каких обязательных элементов состоит таблица?
3. Для чего нужен SQL?
4. Назови три любые реляционные СУБД и кратко опиши, где они применяются.

## День 2. Основы выборки: `SELECT` и `FROM`

**Что изучим:**
- Как получить данные из таблицы с помощью `SELECT`.
- Синтаксис: ключевые слова, идентификаторы, точка с запятой.
- Выбор конкретных столбцов и всех столбцов (`*`).
- Ключевое слово `FROM` для указания источника данных.
- Понятие алиасов для столбцов (`AS`) и как они улучшают читаемость результата.
- Простейшие выражения в `SELECT` (например, `price * 1.2`).

**Ключевые мысли:**
`SELECT` не изменяет данные — он только читает. Порядок столбцов в выводе можно задавать произвольно. Алиасы (псевдонимы) столбцов полезны, когда название должно быть понятным пользователю отчёта. Выражения в `SELECT` позволяют делать вычисления на лету.

```sql
-- Все столбцы
SELECT * FROM products;

-- Конкретные столбцы
SELECT title, price FROM products;

-- Алиас и выражение
SELECT title, price * 0.9 AS discounted_price FROM products;
